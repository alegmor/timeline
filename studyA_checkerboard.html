The checkerboard pattern & the Gaussian filter

Broadband initial noise excites the Nyquist mode; a short, periodic Gaussian blur removes it during the transient and is switched off at coexistence.

source of the artefact The instability is seeded by adding independent uniform noise at every lattice site — a broadband perturbation with energy all the way up to the highest wavenumber the grid can carry. PonD does not damp that grid-scale content, so a standing odd–even oscillation grows at the Nyquist mode

$$ \mathbf{k}_{\mathrm{Nyq}}=(\pi,\pi)\quad\Longleftrightarrow\quad \rho_{ij}\;\sim\;\cos(\pi i)\cos(\pi j), $$

i.e. a value that flips sign every cell — the two-cell checkerboard. It is a known lattice-Boltzmann pathology (odd–even decoupling), not a bug specific to this solver; selective spatial filters are the standard LBM remedy.

the fix — a periodic Gaussian blur A two-dimensional Gaussian smoothing pass is applied to the macroscopic fields every \(\ell\) steps, with periodic boundary conditions,

$$ \rho\leftarrow\mathcal{G}_{\sigma}\!\ast\rho,\qquad u_x\leftarrow\mathcal{G}_{\sigma}\!\ast u_x,\qquad u_y\leftarrow\mathcal{G}_{\sigma}\!\ast u_y. $$

For \(\sigma\lesssim 1\) the kernel's transfer function \(\hat{\mathcal{G}}(\mathbf{k})=e^{-\sigma^2|\mathbf{k}|^2/2}\) is \(\approx e^{-\sigma^2\pi^2}\) at the Nyquist mode — an attenuation of order \(10^{-3}\) at \(\sigma=0.8\) — while long-wavelength structure (\(|\mathbf{k}|\to 0\)) passes essentially untouched. The Gaussian is separable, so the 2-D blur is two cheap 1-D passes.

Pseudocode — how the filter is applied
# ── one-time setup ─────────────────────────────────────────
filter_every ← 5           # cadence: apply every 5 steps
filter_sigma ← 0.8         # Gaussian width, lattice units
filtering_on ← (filter_every > 0) and (filter_sigma > 0)

# ── the filter itself: a periodic 2-D Gaussian blur ────────
function gaussian_smooth(field, sigma):
    return gaussian_filter(field, sigma, bc = periodic)   # mode='wrap'

# ── inside the time loop, AFTER each PonD update ───────────
for step = 0, 1, 2, … while step·dt < tmax:
    (f, u, ρ, P) ← pond_evolve(f, u, ρ, P, …)   # stream + collide + PC solve

    # Maxwell-coexistence convergence test
    if ρ.min ≤ (1+ε)·ρ_v and ρ.max ≥ (1−ε)·ρ_l:
        converged ← true              # freezes the filter from here on

    # Gaussian checkerboard filter — transient only
    if filtering_on and (step mod filter_every == 0) and not converged:
        ρ    ← gaussian_smooth(ρ,   filter_sigma)
        u_x  ← gaussian_smooth(u_x, filter_sigma)
        u_y  ← gaussian_smooth(u_y, filter_sigma)
        # populations f are NOT touched — the next pond_evolve
        # rebuilds equilibrium from the smoothed (ρ, u)
Why it does not contaminate the result: the blur injects a small artificial diffusion, so it runs only during the phase-separation transient and switches off automatically the first time the bulk densities reach coexistence, \(\rho_{\min}\le(1+\epsilon)\rho_v\) and \(\rho_{\max}\ge(1-\epsilon)\rho_l\). The late-time equilibrium morphology — the quantity the benchmark actually tests — is therefore governed entirely by the unmodified PonD dynamics. The populations \(f\) are never blurred directly: only \((\rho,\mathbf{u})\) are smoothed, and the next PonD step rebuilds \(f\) from its equilibrium.
